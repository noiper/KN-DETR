# Phase 2: Adaptive Path Gating (APG) Training Guide

Phase 2 focuses on training the **APG Router**, a lightweight gating module that decides whether a video frame should follow the "Key" path (Full RT-DETR) or the "Non-Key" path (Feature Fusion).

---

## 1. Key Concepts

### Causal Sampling
To align with real-time inference, the training loop strictly selects **preceding** frames as candidate keys.
- **`m` (sampled_keys)**: The number of past frames evaluated for every non-key frame.
- **`allow_same_frame`**: If enabled, the current frame is its own best candidate (identity mapping).

### Multiplicative Target Score
We use a coupled formula to determine the "Ground Truth" quality of a candidate Key:
$$Score = \lambda_{cls} \times L_{cls} \times (1 + \lambda_{conf} \times \max(0, Conf_{Key} - Conf_{NK}))$$
- High loss or a large confidence gap creates a high score.
- The router is trained to predict these scores and select the frame with the lowest degradation.

### Thresholding Methods
Pseudo-labels (BAD=1, GOOD=0) are generated by comparing a candidate's score to a baseline:
1. **`local`**: Baseline = $\min(\text{scores of current candidates})$. Standard for adapting to per-scene difficulty.
2. **`ema`**: Baseline = Global Exponential Moving Average. Stable across batches but ignores per-scene context.
3. **`per_sample`**: Baseline = Score of the **temporally closest** frame. The strictest method, assuming the most recent history is the ideal anchor.

## 2. Training (Phase 2)

### Command
```bash
python tools/phase2_train.py \
    -c configs/kndrtr/phase2_kndetr_mot17.yml \
    -t output/phase1_mot17_level4/checkpoint0019.pth \
    --output_dir output/phase2_result
```

### Key Arguments
| Argument | Meaning | Recommended |
| :--- | :--- | :--- |
| `-c, --config` | Path to the Phase 2 YAML config. | |
| `-t, --tuning` | **Frozen Detector Weights.** Use a trained Phase 1 checkpoint. | |
| `--sampled_keys` | Number of past frames to evaluate as candidates ($m$). | 4 |
| `--beta` | Relative strictness. A score is BAD if it is `beta` times worse than baseline. | 1.1 - 1.2 |
| `--conf_topk` | Number of top queries used to calculate the confidence proxy. | 10 - 20 |
| `--threshold_type` | Method for training labels (`local`, `ema`, `per_sample`). | `local` |
| `--eval_val` | Run pseudo-label validation every epoch. | Enabled |

---

## 3. Evaluation (Stream Simulation)

Evaluation simulates a real video stream where the router makes real-time decisions.

### Command
```bash
python tools/phase2_eval.py \
    -c configs/kndrtr/phase2_kndetr_mot17.yml \
    -w output/phase2_result/apg_best.pth \
    --threshold 0.5 \
    --nk_per_key 10
```

### Key Arguments
| Argument | Meaning | Recommended |
| :--- | :--- | :--- |
| `-w, --checkpoint` | **Trained Router Weights** from Phase 2. | |
| `--threshold` | Probability threshold. If `prob > threshold`, trigger a Key frame. | 0.5 |
| `--nk_per_key` | **Guard Rail.** Maximum consecutive Non-Key frames before forcing a Key frame. | 5 - 10 |
| `--tune_score` | Brute-force search for optimal Key/Non-Key confidence scaling. | Optional |

---

## 4. Troubleshooting Overfitting

If `train_loss` drops but `val_loss` increases:
1. **Lower `beta`**: Your standards might be too high for stable validation scenes.
2. **Lower `conf_topk`**: Averaging too many queries can "wash out" the quality signal.
3. **Check EMAs**: Verify that `train_ema` and `val_ema` in the logs are stabilized around the dataset's average losses.
